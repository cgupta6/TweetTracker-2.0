  • tweet_id – The Twitter-assigned ID of the tweet.
  • catime – The catime index of the tweet.
  • response_type – The encoding format for the results of the request.

SearchAPI.api_support.three_search(username, begin_time, end_time, job_ids, query_object, limit, skip, config, no_limit=False, response_type='JSON', remove_fields={})¶

This is an implementation of the UI-side search found in TweetTracker.

Parameters:
  • username – The user doing the search
  • begin_time – The time to begin looking for tweets
  • end_time – The time to end looking for tweets
  • job_ids – The ids of the jobs we want to take tweets from
  • query_object – The query we are performing
  • limit – The number of results to return
  • skip – The number of results to skip
  • no_limit – If true, ignore the limit parameter and return all results
Returns:

A response either containing search results or an error
