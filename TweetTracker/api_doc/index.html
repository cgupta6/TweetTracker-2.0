TweetTracker API V 0.2 (beta)

Provides a series of endpoints to query data collected from ASU's TweetTracker.

Login

Resource URL

/login

Parameters

Parameter Type Takes Multiple Required Default Description
username String No Yes N/A The username to authenticate as
password String No Yes N/A The password for the provided username

Output

This call returns a JSON document {"login": "success"} with HTTP status code 200 if successful, and a HTTP 400 Bad Request if the request is missing parameters. If the username provided doesn't exist or the password doesn't match the username, the server will respond with an HTTP 403 status code.

If you received a HTTP 200 Success response, the response will have a "session" cookie that is used to authenticate later requests. To make authenticated requests to the API, simply call /login with your credentials, save the "session" cookie, then attach that cookie to all subsequent requests.

Suppose you have an account with the username foo and the password bar. The following is an example of a successful login:

HTTP Method URL Request Body
POST /login { "username": "foo", "password": "bar" } 
{ "login": "success" }

Get Jobs

Resource URL

/api/job

Parameters

The Get Jobs API endpoint doesn't require any parameters -- it will automatically respond with all the public jobs and the jobs that correspond to the user stored in the session cookie. This means that you must first make a call to login and set the session cookie in order to receive your jobs. However, if called with no active session, it will respond with all the public jobs.

Output

The "jobs" property of the output of this API call has the following format:

Property Type Description
catname String The name of the Job
publicflag Integer 1 signifies that this Job is publicly accessible, 0 signifies it is not.
geoboxes Array[Geobox] An array of Geoboxes that the Job is tracking.
creator Integer The user id of the user that created the Job.
keywords Array[String] The keywords that the Job is tracking.
userids Array[UserObj] An array of UserObj that contains information about the Twitter users the Job tracks.
desc String A short description of the Job.
categoryID Integer A unique identifier for the Job.
last_tweet_time Integer The timestamp for when the last tweet in the Job was added.
createtime Integer A timestamp for when the job was created.

Some of the objects have special types that we defined (Geobox, UserObj). Below are the structures of these types.

Geobox
Property Type Description
neLat Double The latitude of the North-East corner of the geobox
neLng Double The longitude of the North-East corner of the geobox
seLat Double The latitude of the South-East corner of the geobox
seLng Double The longitude of the South-East corner of the geobox
swLat Double The latitude of the South-West corner of the geobox
swLng Double The longitude of the South-West corner of the geobox
nwLat Double The latitude of the North-West corner of the geobox
nwLng Double The longitude of the North-West corner of the geobox
UserObj
Property Type Description
userID Integer The user ID assigned to the user by Twitter
screen_name String The screen name of the user on Twitter

The output of this API call is a JSON object containing a "jobs" array. Each element of this jobs array is itself a JSON object representing the job in TweetTracker. Below is an example, where the session cookie has been set to correspond to our admin user, which is truncated:

Example

HTTP Method URL
GET /api/job 
{ "jobs": [
    {
        "catname": "Anti-Maydan",
        "publicflag": 1,
        "geoboxes": [
            {
                "latlngsw": [
                    11.673,
                    102.875
                ],
                "latlngne": [
                    21.35,
                    107.182
                ]
            }, ... ],
        "creator": 5,
        "keywords": [
            "antimaydan",
            ... ],
        "userids": [
            {
                "userID": 2361029106L,
                "screen_name": "anti_maydan"
            }, ... ],
        "desc": "Following pro-Russian propaganda about the Ukraine"
        "categoryID": 352

        "createtime": 1396801085931L
    }, ... ]
}

Edit Job

Resource URL

/api/job/<:jobId>

Parameters

Before making a call to this endpoint, you must be logged in to the user who owns the job with the id <jobId>. jobId should the the id of a job that you retrieved through a get_jobs request. This endpoint expects the following parameters as a json object:

Parameter Type Required Default Description
name String Yes n/a The name we want the job to have.
keywords Array[String] No [] The keywords we want the job to track.
users Array[String] No [] The Twitter screen names of the users we want the job to track.
geoboxes Array[Geobox] No [] The locations we want the job to track.

Output

The output of this API call is a JSON object {"status": "ok"} if the job was edited correctly, or an HTTP error code otherwise.

Example

HTTP Method URL Request Body
PUT /api/job/3 
{
    "name": "NewJobName",
    "keywords": ["Jobs", "New"],
    "users": ["Barack"],
    "geoboxes": [
        {
          "neLat": 42.773,
          "neLng": -70.661,
          "nwLat": 42.773,
          "nwLng": -71.551,
          "seLat": 42.035,
          "seLng": -70.661,
          "swLat": 42.035,
          "swLng": -71.551
        }
    ]
}
 
{"status": "ok"}

Delete Job

Resource URL

/api/job/:id

Parameters

Before making a call to this endpoint, you must be logged in. The user you are logged in must be the owner of the job with the id specified by the :id portion of the URL. This request requires no parameters or body.

Output

The output of this API call is a JSON object {"status": "ok"} if the job was deleted successfully, or an HTTP error code otherwise.

Example

HTTP Method URL Request Body
DELETE /api/job/1 
{"status": "ok"}

Set Crawl

Resource URL

/api/job/:id/set_crawl

Parameters

Before making a call to this endpoint, you must be logged in. The user you are logged in must be the owner of the job with the id specified by the :id portion of the URL. This request has a very simple body JSON object structure.

Parameter Type Required Default Description
crawl Boolean Yes n/a true to crawl, false to not crawl.

Output

The output of this API call is a JSON object {"status": "ok"} if the job was deleted successfully, or an HTTP error code otherwise.

Example

HTTP Method URL Request Body
PUT /api/job/1/set_crawl {"crawl": true} 
{"status": "ok"}

Get Tweets

Resource URL

/api/gettweets

Parameters

Parameter Type Takes Multiple Required Default Description
categoryID Integer Yes Yes N/A Category IDs that you want to retrieve. These are the "categoryID" field as returned by the getJobs call.
start_time Integer No No Two days before end_time The start time of the query as a Unix timestamp without milliseconds. Fetches tweets created after this time.
end_time Integer No No Current Time The end time of the query as a Unix timestamp without milliseconds. Fetches tweets created before this time.
keyword String Yes No N/A Searches for this keyword in the text. If multiples are provided, treated as logical "or".
user String Yes No N/A Searches for tweets authored by this user in the text. If multiples are provided, treated as logical "or".
geobox List of floats Yes No N/A Of the form "Southwest Longitude, Southwest Latitude, Northeast Longitude, Northeast Latitude". If multiples are provided, treated as logical "or".
skip Integer No No 0 Skip the first tweets returned by the query.
limit Integer No No 1000 Limits the query to this many documents. Maximum value is 1000.

  • Return tweets from category 370 containing the word "scrimmage" or "picture": /api/gettweets?categoryID=370&keyword=scrimmage&keyword=picture

Get Trendlines

Resource URL

/api/gettrendline

Parameters

Parameter Type Takes Multiple Required Default Description
categoryID Integer Yes Yes N/A Category IDs that you want to retrieve. These are the "categoryID" field as returned by the getJobs call. Separate trendlines are provided for each category
start_time Integer No No Jan. 1, 1970 The start time of the query as a Unix timestamp without milliseconds. Constructs trendlines for tweets created after this time.
end_time Integer No No Current Time The end time of the query as a Unix timestamp without milliseconds. Constructs trendlines for tweets before this time.
keyword String Yes No N/A Searches for this keyword in the text. If multiple are provided, separate trendlines are provided for each keyword in each category.
granularity Integer (≥ 1) No No 1 Time intervals at which the tweets are sampled and aggregated. The lowest granularity is 1 hour and integer multiples of 1 hour are accepted.

Box Search

Resource URL

/api/three_search

Description

Box Search is the backend implementation of the search provided on Search/Export.

Parameters

Parameter Type Takes Multiple Required Default Description
begin_time Integer No Yes N/A The start time of the query as a Unix timestamp without milliseconds. Fetches tweets created after this time
end_time Integer No Yes N/A The end time of the query as a Unix timestamp without milliseconds. Fetches tweets created before this time.
job_ids Integer Yes Yes N/A The list of job ids to search from
query BoxQuery No Yes N/A The query to execute against the database.
limit Integer No No 50 The number of results to return from the query.
skip Integer No No 0 The number of results to skip before returning the results in the response.
response_type { "XML", "JSON" } No No "JSON" The type of encoding you want for your response, either JSON or XML.

BoxQuery

This is a description of the BoxQuery object that is part of the request parameters. The logic is to take tweets matching box1 AND box2 AND NOT box3.

Property Type Required Description
box1 Array[BoxParam] Yes The first box of parameters.
box2 Array[BoxParam] Yes The second box of parameters.
box3 Array[BoxParam] Yes The third box of parameters. This box is different in that it contains parameters that must NOT be met.

Box Param

This is a description for the BoxParam object that is part of the BoxQuery.

Property Type Required Description
type { "user", "key", "geo" } Yes This property describes what type of parameter this is.
data String | Geobox Yes The actual parameter. If type is user or keyword then this is just a string of the screen name or keyword. If type is "geobox" then this is a Geobox.

Output

This call returns a JSON list of data pulled from tweets for display in the Search/Export tab. The returned object has the following format.
Name Type Description
count Integer This property states how many tweets match the query. This is useful for pagination.
tweets Array[SearchTweet] The actual tweets that match the query. See the SearchTweet object for more info

SearchTweet

The format of tweets returned from the search API
Name Type Description
geo Boolean Boolean indicating if the tweet is geotagged.
text String The actual text of the tweet.
timestamp Integer The UNIX timestamp of when the tweet was published.
screen_name String The screen name of the user who submitted the tweet.
id String The unique id of the tweet as assigned by Twitter.
job_id Integer The id of the job that this tweet matched.
lat Real The lattitude of the tweet, if available
lng Real The longitude of the tweet, if available
catime String A special indexing field for the tweets database used to make subsequent requests to particular endpoints.
lang Integer Twitter's assigned language for the tweet
entities TwitterEntities The entities attached to the Tweet by Twitter (see the link for more info).
user TwitterUser The user object attached to the Tweet by Twitter (see the link for more info).

Get Locations

Resource URL

/api/entities/locations

Description

This endpoint retrieves the lat/lng coordinates of geotagged tweets from the provided jobs in a set time period. You must be authorized to read from all of the jobs that are passed in the job_ids parameter.

Parameters

Parameter Type Required Accepts Multiple Default Description
categoryID Integer Yes Yes N/A The jobs to get locations from.
begin_time Integer Yes No N/A The UNIX timestamp of the beginning of the time period to get locations from.
end_time Integer Yes No N/A The UNIX timestamp of the end of the time period to get locations from.

Output

The response to a successful request has the following format:

Name Type Description
locations Array[Location] The requested Tweet locations

Location
Name Type Description
lat Real The lattitude of the tweet, if available
lng Real The longitude of the tweet, if available
index String This is an indexing string that can be used in a subsequent call to get_tweet to retrieve more information about this Tweet.

Example Call

    },{
        "index":"492927840862896128-661406358130",
        "lat":36.620317,
        "lng":36.210502
    }, ...
]}
